# 代码民族志与互联网治理中的权力物质性

**英文原标题**：Code Ethnography and the Materiality of Power in Internet Governance
**作者**：Fernanda R. Rosa
**来源**：Qualitative Sociology，2022
**DOI**：10.1007/s11133-022-09517-3
**关键词**：代码民族志(Code Ethnography)、数字民族志(Digital Ethnography)、数字不平等(Digital Inequalities)、互联网互联基础设施(Internet Interconnection Infrastructure)、互联网交换点(Internet Exchange Points, IXP)、互联网治理(Internet Governance)、边界网关协议(Border Gateway Protocol, BGP)

---

### 核心论点

本文提出并阐述了「代码民族志」(code ethnography)这一方法论创新，将其定义为一种以代码(code)作为社会-技术行动者(socio-technical actor)进行民族志研究的方法，旨在揭示数字基础设施中代码的社会、政治和经济动态。Rosa将此方法应用于互联网互联(internet interconnection)动态的研究，以边界网关协议(BGP)作为分析对象，以全球最大的两个互联网交换点——德国法兰克福的DE-CIX和巴西圣保罗的IX.br——作为田野调查点。

研究的核心发现揭示了全球北方(global North)与全球南方(global South)之间在信息流动层面的深层不平等，以及在互联基础设施层面此前未被充分认识的权力集中现象。具体而言，DE-CIX作为国际节点吸引了70%的国际网络，而IX.br则以96%的本国网络为主，呈现出明显的不对称性。DE-CIX宣告的唯一前缀(unique prefixes)数量是IX.br的近五倍（496,909 vs. 105,960），这在物质层面解释了为何巴西互联网服务提供商选择在德国交换流量——因为连接DE-CIX能获得更短的AS_PATH和更多的地址可达性。

Rosa通过三个核心概念框架来阐释代码民族志：「代码装配」(code assemblage)将BGP置于互联网政治经济的行动者网络中；「代码素养」(code literacy)讨论研究者与代码的互动能力；「代码物质性」(code materiality)则强调BGP以其自身语言所传达的信息——即代码的逻辑维度与物理维度不可分割，共同构成了互联网信息流通基础设施中「巨型互联网节点」(giant internet nodes)的形成和「基础设施相互依赖」(infrastructural interdependencies)。文章最终论证，代码所共同生产(co-produce)的权力不平等嵌入在互联网基础设施之中，反映了数字殖民性(coloniality of power)的当代形式。

### 理论框架

本文立足于基础设施研究(infrastructure studies)、科学技术研究(STS)和互联网治理(internet governance)的交叉领域。核心理论资源包括：Deleuze与Guattari的「装配」(assemblage)概念，用以理解代码作为连接人类与非人类行动者的异质性集合；Susan Leigh Star的基础设施民族志(ethnography of infrastructure)传统；George Marcus的多点民族志(multi-sited ethnography)方法；Latour的行动者网络理论(ANT)中的对称性原则(principle of symmetry)；以及Manuel Castells关于网络社会中「切换与编程的具体手段决定权力形式」的论述。此外，文章还借鉴了Wendy Chun关于软件与硬件不可分离的批判性代码研究，以及Quijano的殖民性理论(coloniality of power)来分析南北数字依赖关系。

### 分节摘要

#### 1. 引言(Introduction)

Rosa从全球数据流量的爆发性增长切入，指出互联网上超过一泽字节(zettabyte)的数据按照「包交换」(packet-switching)设计在多条路径间传输，而路由决策发生在设备、协议与网络中介的互动中。这些决策对公众而言几乎不可见，但具有深刻的社会、政治和经济意涵。文章提出将「代码」(code)纳入讨论，作为理解互联网信息流的新途径。Rosa回顾了自己的田野经历：从2016年在巴西NIC.br的基础设施民族志开始，最初聚焦于人、服务器、交换机等物理行动者，直到在DE-CIX的田野工作中通过Looking Glass应用接触BGP数据，才意识到代码作为不可回避的行动者的重要性。Marcus的多点民族志理念和Wendy Chun「软件与硬件不可分离」的观点为这一方法论转向提供了理论支撑。

#### 2. 论政治：代码作为装配的一部分(On Politics: The Code as part of an Assemblage)

本节阐述BGP作为互联网路由器之间的通信语言，如何在逻辑和经济层面塑造互联网互联。Rosa详细解释了「对等互联」(peering)与「转接」(transit)两种基本互联模式，以及AS_PATH属性如何决定最短路由。IXP作为网络汇聚的物理节点，物化(materialize)了互联基础设施最重要的物理维度。研究数据显示，全球五分之一的路由经过IXP。Rosa进而讨论了巴西IXP生态系统的形成——从1990年代开始，出于维护国家主权的需要，将原本在美国或北美中转的流量转移到本国领土。然而尽管巴西拥有全球最大的公共IXP生态系统（超过30个节点），77%的巴西最常访问域名仍在美国解析，84%的访问请求必须经过美国。通过一位BGP专家的访谈，Rosa揭示了「自治系统」(autonomous system)层面的主权悖论：每个自治系统在技术上是主权的，但数据一旦发出，接收方即拥有对数据的完全控制权。

#### 3. 论技术：代码素养(On the Technique: Code Literacy)

Rosa将代码素养(code literacy)定义为个人与代码互动以回应社会需求的能力条件，它根植于多种技能和生活经验的积累。她强调代码素养与数字素养(digital literacy)的区别在于其「情境性」(situated)和「功能性」(functional)特征——不依赖于编码或计算机编程技能本身，而是一种「社会实践」(social practices)。Rosa详细列出了BGP代码素养的实践方法清单（表1），包括：法律与技术文档分析、文献分析、公共数据库分析、代码采集、深度访谈、持续对话和参与观察。在代码采集方面，她与一位在全球多个IXP工作过的网络工程师合作，量化了BGP的两个字段、连接网络和来源网络，并与世界银行和联合国数据库交叉分析，以操作化「全球北方/南方」的概念。Rosa还讨论了私有化管理的代码所带来的数据获取障碍——IX.br的数据可从其路由服务器下载，而DE-CIX的数据则需通过Packet Clearing House间接获取。

#### 4. 论物质性：代码的言说(On Materiality: What the Code has to Say)

本节是文章的核心分析部分。Rosa通过BGP「前缀宣告」(prefix announcement)的可视化表示，展示了IXP路由数据的解读方法——包括前缀、路由器IP地址、AS_PATH等字段如何物化互联网络之间的客户-提供商关系。比较分析显示：IX.br圣保罗虽然拥有近两倍于DE-CIX的对等互联自治系统数量（1,720 vs. 879），但IX.br主要是国家级节点（96%为巴西网络），而DE-CIX则高度国际化（70%为国际网络）。DE-CIX吸引的南方网络比例（23.2%，204个网络）远高于IX.br吸引的北方网络比例（3.3%，57个网络），揭示了从全球南方到全球北方的持续信息流动。更关键的是，DE-CIX宣告的唯一前缀数量是IX.br的近五倍（496,909 vs. 105,960），这从物质层面解释了巴西ISP选择在德国交换流量的经济逻辑——连接DE-CIX意味着更多的地址可达性和更优的AS_PATH选项。Rosa将此称为「基础设施相互依赖」(infrastructural interdependency)，认为这体现了嵌入数字基础设施中的殖民性权力形式。

#### 5. 结论(Conclusions)

Rosa总结代码民族志的成果应被理解为高度动态环境的一个快照。数字民族志为揭示与权力、依赖和不平等相关的经典社会学问题开辟了空间。互联网上的权力集中超越了我们在平台研究中所见——内容提供商的力量和BGP语法在基础设施层面产生了针对全球南方的不平等。Rosa呼吁重新思考公共政策：如果巴西公民的数据因自治系统的决策流向德国或其他地方，公共利益能否优先于私人利益？代码民族志通过将BGP作为行动者置于其语境中来回应这类问题。文章最后强调，代码民族志者应同时关注逻辑维度和物理维度，因为物理基础设施（如IXP）共同生产并阐明逻辑组件（如BGP），物理制品也可作为获取代码的入口。

### 关键概念

**代码民族志(Code Ethnography)**：一种将代码视为社会-技术行动者进行民族志研究的方法论创新。它不同于研究代码周围人群的传统数字民族志，而是直接以代码本身为对象，通过代码装配、代码素养和代码物质性三个维度展开。Rosa将其定位为技术领域（计算机科学与工程）和社会科学与人文学科之间的「桥梁」，借鉴了Gloria Anzaldua的边界概念。该方法不依赖编程技能，而是强调情境化的社会理解能力。

**代码装配(Code Assemblage)**：借自Deleuze和Guattari的「装配」(assemblage)概念，指代码与其运行环境中的人类和非人类行动者共同构成的异质性关系网络。在本文中，BGP的装配包括网络工程师、互联网社群专家、国家、流量交换点、转接提供商、路由器、交换机和互联网用户等。这一概念拒绝将代码视为孤立对象，强调其属性和特征必须在关系和语境中被理解，呼应了Latour关于「有目的的行动和意向性」既非物体也非人类的属性，而是Foucault所谓「装置」(dispositifs)的属性的论述。

**代码素养(Code Literacy)**：个人与代码互动以回应社会需求的能力条件，建立在多种技能和生活经验的基础上。Rosa区分了代码素养与编程技能：前者是情境性的(situated)和功能性的(functional)，类似于教育领域中功能性素养与形式性素养的区分。开发BGP代码素养涉及一系列实践方法（文档分析、数据库分析、代码采集、访谈、参与观察等），核心在于理解代码「在做什么」(mechanology)而非仅仅「是什么」(ontology)。

**巨型互联网节点(Giant Internet Nodes)**：Rosa用来描述像DE-CIX法兰克福这样的大型IXP的概念，它们因内容提供商的物理存在而获得巨大的吸引力，不断吸引更多网络连接，形成权力集中效应。这些节点的社会、经济和政治影响需要特别关注，因为它们塑造了数据流通基础设施中的不平等格局，维持了全球南方对全球北方基础设施的依赖。

**基础设施相互依赖(Infrastructural Interdependency)**：描述全球北方巨型互联网节点与全球南方互联网服务提供商之间不对称关系的概念。研究数据显示，众多巴西ISP连接DE-CIX正是因为南北之间在连接性水平上的差距——南方缺乏足够的连接性和内容提供商资源，推动其网络持续向北方巨型节点集中，后者因此变得更大，形成自我强化的不平等循环。Rosa将此与Quijano的殖民性理论联系起来，认为这体现了嵌入数字基础设施中的殖民性权力形式。

### 与技术、AI、媒介与具身化的关联

1. **数字基础设施的物质性与权力**：本文深刻揭示了互联网基础设施并非中立的技术管道，而是嵌入了权力关系的物质-逻辑装配。BGP代码的语法（特别是AS_PATH属性）物化了经济逻辑和地缘政治不平等，这为理解技术的物质性(materiality)提供了基础设施层面的实证支撑。代码的物理维度（IXP、光缆、路由器）与逻辑维度（BGP协议、路由表）不可分割，共同构成了数据流动的权力地形。

2. **代码作为社会-技术行动者**：Rosa将代码从工具性存在提升为具有能动性(agency)的行动者，这一视角与AI研究中关于算法能动性的讨论直接相关。BGP作为「政治化」(politicking)的协议，其路由决策不仅是技术行为，也是经济和政治行为。这一分析框架可延伸至AI算法如何在其代码逻辑中嵌入并再生产社会不平等。

3. **方法论创新与数字民族志**：代码民族志作为方法论贡献，为研究技术和AI系统提供了一种不依赖源代码访问权限的分析路径。Rosa展示了如何通过代码素养（一种情境化的、具身化的能力）来「阅读」技术系统的输出，这对研究不透明的AI系统（如黑箱算法）具有方法论启示。

### 价值评估

**相关性评级：★★★（高度相关）**

**理论贡献**：本文对技术、AI、媒介与具身化研究的理论价值在于三个层面：（1）提出代码民族志作为研究不透明技术系统的方法论创新；（2）以实证数据揭示互联网基础设施层面的权力不平等，超越了平台层面的分析；（3）将代码素养概念化为一种情境化的、具身化的社会能力，而非单纯的技术技能，为理解人与技术系统的互动提供了新视角。

**局限性**：文章主要聚焦于两个IXP的比较分析，时间点为2019年，数据的时效性有限。对代码民族志方法的可复制性和推广性讨论不足——其他研究者如何在不同技术语境中应用这一方法仍不明确。此外，文章对BGP路由决策的分析主要停留在宏观结构层面，对微观互动过程（如网络工程师日常的路由配置决策）的分析相对薄弱。

**推荐追踪**：
1. Rosa的后续研究及其关于全球南方数字主权(digital sovereignty)的系列工作
2. 算法民族志方法论的最新进展，特别是Seaver、Christin和Jaton的工作
3. 批判性代码研究(critical code studies)领域，特别是Marino(2020)的方法论框架
